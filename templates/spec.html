Hopper Spec

The following link relationships and namespaces are used in the context of the Web Host Metadata Spec.

https://hopper.at/rel/link

A Hooper link relation type.

This link relationship URI is used by Hopper to find link structures that include a template used to transform AT-URI structures.

See also: https://atproto.com/specs/at-uri-scheme

https://atproto.com/ns/authority

A filtering property for authority matching.

When present, the link only matches AT-URIs with the specified authority (handle or DID).

https://atproto.com/ns/collection

A filtering property for collection matching.

When present, the link only matches AT-URIs with the specified collection value. When omitted, the link matches URIs regardless of their collection (wildcard behavior).

https://atproto.com/ns/rkey

A filtering property for record key matching.

When present, the link only matches AT-URIs with the specified record key.

Templates

This project uses the Restricted AT URI Syntax.

AT-URI        = "at://" AUTHORITY [ "/" COLLECTION [ "/" RKEY ] ]

AUTHORITY     = HANDLE | DID
COLLECTION    = NSID
RKEY          = RECORD-KEY

URI templates can use the following variables:

  1. {authority} - The authority portion of the AT-URI (handle or DID)
  2. {collection} - The collection NSID
  3. {rkey} - The record key

This deviates from spec as typically only the {uri} variable is supported.

Integration Notes

When a Web Host Meta structure is parsed, the following rules are applied:

  1. Only links with the rel https://hopper.at/rel/link are used.
  2. Only links with a template attribute are used.
  3. The template must have the same hostname as the server.
  4. If the properties contain namespace properties (https://atproto.com/ns/authority, https://atproto.com/ns/collection, or https://atproto.com/ns/rkey), the AT-URI must match those filters.

Optional, when serving the /.well-known/host-meta.json file, use the recommended application/jrd+json content type.

Example /.well-known/host-meta.json

{
  "links": [
    {
      "rel": "https://hopper.at/rel/link",
      "template": "https://smokesignal.events/{authority}/{rkey}",
      "properties": {
          "https://atproto.com/ns/collection": "community.lexicon.calendar.event"
      }
    }
  ]
}

See also: https://smokesignal.events/.well-known/host-meta.json
